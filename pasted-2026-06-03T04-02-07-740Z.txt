ETAPA 2 — RLS COMPLETO
==============================================================

Para CADA tabela que tem agency_id, garantir que existem políticas:

FUNÇÃO helper (criar se não existir):
CREATE OR REPLACE FUNCTION get_my_agency_id() RETURNS uuid AS $$
  SELECT agency_id FROM profiles WHERE id = auth.uid() LIMIT 1;
$$ LANGUAGE sql STABLE SECURITY DEFINER;

CREATE OR REPLACE FUNCTION is_agency_member(aid uuid) RETURNS boolean AS $$
  SELECT EXISTS(SELECT 1 FROM profiles WHERE id = auth.uid() AND agency_id = aid);
$$ LANGUAGE sql STABLE SECURITY DEFINER;

CREATE OR REPLACE FUNCTION has_role(aid uuid, r text) RETURNS boolean AS $$
  SELECT EXISTS(SELECT 1 FROM user_roles WHERE user_id = auth.uid() AND agency_id = aid AND role = r);
$$ LANGUAGE sql STABLE SECURITY DEFINER;

POLÍTICA PADRÃO para tabelas multi-tenant (aplicar em todas):
- SELECT: agency_id = get_my_agency_id()
- INSERT: agency_id = get_my_agency_id()
- UPDATE: agency_id = get_my_agency_id()
- DELETE: agency_id = get_my_agency_id() AND NOT has_role(agency_id, 'agent_viewer')

EXCEÇÕES:
- contracts: DELETE bloqueado para todos (apenas super_admin via service role)
- audit_log: apenas INSERT permitido
- contracts após signed_at: UPDATE bloqueado pelo trigger

CLIENTES (portal): usuários com role 'client' só veem:
- trips onde id IN (SELECT trip_id FROM trip_passengers WHERE client_id = auth.uid())
- proposals onde client_id = (SELECT id FROM clients WHERE user_id = auth.uid())
- financial_records onde trip_id IN (suas viagens)
- support_tickets onde client_id = (SELECT id FROM clients WHERE user_id = auth.uid())
