# Protocolo de Verdade e Evidências

## 1. Princípios
Nenhuma afirmação ou conclusão técnica é aceita como verdade automática. Todo diagnóstico ou correção deve documentar a árvore de causa raiz do DOM até o RLS e provar sua eficácia no runtime.

## 2. Critérios de Prova
- Para visual: Capturas de tela ou styling computado.
- Para funcionalidade: Logs de requests/respostas HTTP ou mutações reais persistidas no banco.
