# RELATO DO USUÁRIO - INCIDENTE DE BOTÕES DE AÇÃO CONTEXTUAIS VAZIOS

## 1. Descrição do Relato (Hipótese)
O usuário relata a ausência e invisibilidade de botões e controles de ação contextuais primários em diversos módulos e tabelas do sistema (como botões de criação rápida: "Novo Lead", "Novo Cliente", "Nova Cotação").

## 2. Comportamento Esperado
* Existência de botões e controles nítidos para todas as ações operacionais primárias do módulo correspondente.
* As ações devem funcionar e ser integradas ao design do cabeçalho sem quebras ou sobreposição.
