## Detalhamento de Desvios Encontrados

1. **Brand Kit & localStorage:** O cache em `localStorage` é isolado por `agency_id`, porém não há mecanismos de invalidação no logout. O design de um tenant permanece no cache local indefinidamente, permitindo vazamento visual e de dados corporativos (fontes, logos) se outro usuário acessar o sistema na mesma máquina.
2. **Onboarding Typing:** A exclusão dos comentários `@ts-ignore` foi mascarada pela inserção de uma conversão para `as any` ao injetar o objeto retornado de `defaultHours()` no parâmetro `_business_hours` da RPC.
3. **Upload de Pix:** O bucket `payment-receipts` foi configurado como **público** e as políticas RLS anexadas permitem leitura e inserção aberta para qualquer cliente anônimo, criando um vazamento geral de PII (comprovantes contendo nomes, CPFs, chaves PIX e valores).
4. **Persistência OCR:** O botão "Confirmar e persistir dados" não possui bloqueio de clique durante o salvamento, e as inserções não são transacionais. Se o usuário clicar repetidamente, múltiplas duplicatas dos contatos e produtos do fornecedor serão salvas no banco de dados.
5. **Omnichannel de Suporte:** Embora a Edge Function `gmail-send` dispare e-mails, o sistema não possui recepção nem sincronização de respostas. O app não ouve webhooks nem tem crons para rodar `gmail-sync`. É apenas um envio unidirecional de e-mails, falhando na premissa de omnichannel.
6. **Snapshot de Contratos:** O walkthrough afirma salvar os snapshots em uma tabela normalizada `trip_contracts`, mas a tabela em uso no banco e no código é `contracts`. O campo `clause_snapshot` recebe um JSONB, porém a política RLS da tabela permite atualização ilimitada de registros já assinados, tornando o contrato mutável pós-assinatura.
7. **Rooming List:** A alocação visual atualiza o array de passageiros inteiramente no cliente e salva por cima do campo `passengers` (JSONB) na tabela `boarding_rooming_list`. Se dois operadores trabalharem na mesma viagem simultaneamente, um sobrescreverá o outro, perdendo atualizações silenciosamente (Lost Update).
