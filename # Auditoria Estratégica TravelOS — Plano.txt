# Auditoria Estratégica TravelOS — Plano de Reconstrução Premium

**Modo:** Plano. Nada será implementado até sua aprovação explícita.
**Base:** leitura direta de 90 rotas, 47 migrations, 45 tabelas, 13 buckets, 28 RPCs, 3 shells, design system, PRDs colados (4.6k linhas) e documentação interna. Nenhuma afirmação sem evidência.

---

## 1. RESUMO EXECUTIVO HONESTO

TravelOS **não é protótipo** — tem 90 rotas reais, 45 tabelas, 28 RPCs, OCR via Edge Function, contratos com WORM trigger, magic-link público, multi-tenant funcional, design tokens sólidos e dark mode completo. **Mas também não é premium ainda.** Tem 4 falhas estruturais críticas, 10 rotas CRUD raso, 6 tabelas órfãs, 9 componentes mortos, ErrorState ausente, search global fantasma e dois módulos inteiros (Plans, Policies, Brand global) **serializados como JSON dentro da tabela `api_keys`** — um anti-padrão grave que precisa ser desfeito.

**Classificação geral por bloco:**

| Bloco | Classe | Justificativa |
|---|---|---|
| Auth + Onboarding | SaaS utilizável | Magic link, invites, OAuth — completos |
| CRM | **Quebrado / Backend oculto** | `leads`/`lead_stages`/`lead_activities` têm RLS ativa **sem nenhuma policy** → SELECT direto retorna `[]` silenciosamente |
| Cotações/Propostas | SaaS premium | Editor, preview, OCR, capa — sólido |
| Contratos | SaaS premium | WORM, certificate, verify público, magic link |
| Vouchers | SaaS utilizável | OCR via Edge Function funciona; falta gerar story 9:16 prometido |
| Embarques | SaaS utilizável | Kanban com persistência, mas cores hardcoded e drag/drop simples |
| Financeiro/DRE/Caixa | SaaS utilizável | RPCs reais, paginação correta |
| Roteiros em grupo | SaaS utilizável | Refatorado recentemente; falta seat-map editor 2D |
| Vistos / Corporativo | CRUD raso | Lista + sheet, sem timeline, sem documentos |
| Suporte | SaaS utilizável | SLA trigger + ticket_messages OK; falta escalonamento e KB sugerida |
| Clientes (360) | CRUD raso | Sem timeline real, sem merge de duplicados, sem soft-delete habilitado em UI |
| Portal público / CMS | SaaS utilizável | Blocks JSONB renderizando, mas **sem versionamento, sem preview, sem sanitização HTML** |
| Blog | CRUD raso | Editor existe; falta publicação agendada, tags reais, relacionados, sanitização |
| Minha Empresa | SaaS utilizável | Cobre quase tudo do spec; falta integração de publicação pública |
| Admin Master Global | **Encenação visual em 5 sub-rotas** | Plans/Policies/Brand serializados em `api_keys`; Travelers/Trips/Agents/Audit/Contracts são listas read-only com `.limit(200)` sem paginação |
| Portal do cliente | CRUD raso | Coupons/Giftcards listam tudo sem filtrar por cliente; trips read-only |
| Mobile (`/m/*`) | SaaS utilizável | `m.checkin` quebrado (TODO no código); resto OK |
| Design System | SaaS premium (tokens) / Genérico (componentes premium ausentes) | Faltam Timeline, Kanban genérico, Wizard, ⌘K, ErrorState, StatCard |
| Supabase / segurança | **Risco alto** | 4 tabelas sem policies, 3 tabelas com `USING (true)` público, bucket público sem filtro, 40 tabelas sem `deleted_at` |

**Veredicto:** O sistema está num platô intermediário. Tem ossatura para virar premium em ~9 fases, mas exige **primeiro um saneamento de segurança e arquitetura**, antes de qualquer feature nova.

## 4. VERIFICAÇÃO DE REALIDADE — PRINCIPAIS GAPS POR MÓDULO

### 4.1 CRM (🔴 backend oculto)
- Tabelas `leads`, `lead_stages`, `lead_activities` têm RLS habilitada **sem nenhuma policy**.
- Frontend faz `supabase.from('leads')` em 6 arquivos — todas retornam vazio para `authenticated`. Funciona só via RPC `get_crm_leads`.
- Não é kanban — é lista. PRD pede Kanban drag/drop.
- Sem timeline visual de atividades. Sem estágios customizáveis por UI.

### 4.2 Admin Master (🔴 encenação visual)
- `admin/plans` → grava planos como JSON em `api_keys` com `provider='__platform_plans__'`. **Não é tabela real.** Tabela `plans` existe vazia/não usada.
- `admin/policies` → conteúdo dos Termos e Política de Privacidade hardcoded como string no `.tsx`. Não usa tabela `policy_documents` real para versionar.
- `admin/brand` → idem, JSON em `api_keys`. Tabela `global_settings` existe mas órfã.
- `admin/agents`, `/travelers`, `/trips`, `/contracts`, `/audit` → `SELECT … LIMIT 200`, read-only, sem paginação, sem filtros úteis, sem ações (não revoga role, não bloqueia agência, não exporta).

### 4.3 Portal CMS / Blog (🟡)
- Renderiza blocks JSONB sim, mas **sem preview live**, sem **versionamento** (`policy_documents` tem versão; `portal_pages` não), sem **sanitização HTML** em `custom_html`/RichText → risco XSS.
- Blog sem **agendamento (`scheduled_at`)**, sem **tags estruturadas** (provavelmente array text), sem **posts relacionados**, sem **sitemap dinâmico**.

### 4.4 Clientes 360 (🟡 CRUD raso)
- `clients.deleted_at` existe, mas UI não expõe soft delete nem restore.
- Sem timeline (trips + proposals + contracts + vouchers + payments num único feed).
- Sem merge de duplicados.
- Sem mapa de consentimentos integrado.

### 4.5 Embarques (🟡)
- Kanban funciona, mas cores de status hardcoded (`#94A3B8`, `#F59E0B`, `#3B82F6`, `#10B981`) em vez de tokens.
- Sem timeline operacional, sem alertas automáticos (X dias antes do embarque).
- Drag/drop simples, sem persistência de ordem dentro da coluna.

### 4.6 Suporte (🟡)
- Thread + SLA trigger OK.
- **Sem escalonamento** automático no estouro de SLA, **sem sugestão de KB**, **sem comentário interno** (note vs reply), sem CSAT pós-fechamento.

### 4.7 Portal do cliente
- `client.coupons` lista **todos** os cupons ativos da plataforma sem filtrar por agência do cliente.
- `client.giftcards` lista read-only, sem comprar/resgatar.
- `client.trips` read-only, sem tracking de status visual rico.

### 4.8 Mobile `/m/*`
- `m.checkin.$token.tsx` **quebrado** — comentário no código admite que RLS anon nega e fallback RPC não foi implementado.

### 4.9 Mídia / Storage
- `agency-media` bucket tem policy `SELECT TO public USING (bucket_id='agency-media')` **sem filtro de pasta/agência** → qualquer anon lista/baixa de qualquer agência.

### 4.10 Segurança Supabase (riscos críticos)
1. `user_roles` sem policies → falha silenciosa de leitura
2. `leads`/`lead_stages`/`lead_activities` sem policies → CRM bloqueado por padrão
3. `company_profiles` SELECT público sem filtro → vaza CNPJ/endereço de todas agências
4. `global_settings` SELECT público → vaza configurações se houver dado sensível
5. `agency-media` storage sem filtro de agência
6. `audit_log` global visível para todos os membros autenticados
7. Várias `SECURITY DEFINER` sem `SET search_path = public`
8. 40 de 45 tabelas sem `deleted_at` (sem retenção/LGPD)
